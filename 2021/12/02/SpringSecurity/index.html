思路

要根据特定的数据库表模式来获取用户的权限

role 表

以上是 user ，role 。user_role 的关系表。 管理员user id ‘1’ 对应了 role id ‘1’ 而role id = 1的 对应的keyword是 ROLE_ADMIN 对应是管理员身份的角色

permission 表

而role 的表跟 permission表 的关系是多对多 的关系， 所以得用中间表 然后 通过两个循环获取权限给Spring Security 组件中 通过多表查询的方式获取到用户的权限信息

例如：在删除表中设置权限，加上@PreAuthorize(“hasAnyAuthority(‘CHECKITEM_DELETE’)”)
